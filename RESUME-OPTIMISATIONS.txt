═══════════════════════════════════════════════════════════════
  🚀 UTILISATION
═══════════════════════════════════════════════════════════════

ÉTAPE 1 : CONFIGURER L'IP (1× seulement)
──────────────────────────────────────────────────────────────

Terminal :
  $ ipconfig getifaddr en0
  
Éditez : capacitor.config.ts (ligne 13)
  url: 'http://VOTRE_IP:3000'
  
Synchronisez :
  $ npm run mobile:sync

──────────────────────────────────────────────────────────────

ÉTAPE 2 : LANCER LE SERVEUR
──────────────────────────────────────────────────────────────

Terminal 1 (gardez-le ouvert) :
  $ npm run dev

──────────────────────────────────────────────────────────────

ÉTAPE 3 : LANCER L'APP
──────────────────────────────────────────────────────────────

Terminal 2 :
  $ npm run mobile:run

Ou via Android Studio :
  $ npm run mobile:open
  → Run → Run 'app'

═══════════════════════════════════════════════════════════════
  ⚠️  IMPORTANT À SAVOIR
═══════════════════════════════════════════════════════════════

→ L'app nécessite un SERVEUR NEXT.JS actif

→ Téléphone et ordinateur doivent être sur le MÊME RÉSEAU
→ Le serveur (npm run dev) doit TOUJOURS tourner

Pourquoi ? L'app utilise :
  • API routes côté serveur
  • Base de données SQLite locale (better-sqlite3)

  🐛 DÉPANNAGE RAPIDE
═══════════════════════════════════════════════════════════════

PROBLÈME : L'app ne se connecte pas
SOLUTION :
  1. Vérifier que npm run dev tourne
  2. Vérifier l'IP dans capacitor.config.ts
  3. Vérifier le WiFi (même réseau)
  4. Tester http://VOTRE_IP:3000 dans le navigateur mobile

──────────────────────────────────────────────────────────────

PROBLÈME : Modifications non visibles
SOLUTION :
  $ npm run mobile:sync
  Puis rebuild dans Android Studio

──────────────────────────────────────────────────────────────

PROBLÈME : Page blanche
SOLUTION :
  1. Vérifier les logs dans Android Studio (Logcat)
  2. Ouvrir chrome://inspect dans Chrome
  3. Vérifier la connexion serveur
